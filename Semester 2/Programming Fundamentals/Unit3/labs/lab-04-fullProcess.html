Problem Statement

A program is required to prompt the user for a first name and a last name. The program will create a greeting that starts with "Hi, " followed by the first name, followed by a space, followed by the last name. The application will then display the greeting on the page.

1. Nouns and Verbs

  • Copy the above problem statement and paste it below so it appears in the black box when you refresh the page, underneath the words PROBLEM STATEMENT.
  • Identify the nouns and verbs in the problem statement and write them in the appropriate locations so they appear in the black box below when you refresh the page.
PROBLEM STATEMENT:
A program is required to prompt the user for a first name and a last name. The program will create a greeting that starts with "Hi, " followed by the first name, followed by a space, followed by the last name. The application will then display the greeting on the page.

Nouns: firstName, lastName, greeting

Verbs: prompt, display

2. Defining Diagram

  • Convert the nouns and verbs into variable names and processes and create a defining diagram.
  • Make your variable names valid JavaScript names that conform to the course JavaScript Coding Standards.
  • Write down the processes with very short statements, 2 or 3 words. Remember, this is not pseudocode. This is just the process.
INPUTS:
  firstName
  lastName
  greeting

PROCESSING:
  prompt firstName
  prompt lastName
  calculate greeting
  output greeting
  

OUTPUTS:
  output greeting

3. Solution Algorithm

  • Create a solution algorithm (using Pseudocode) based on the defining diagram.
  • Follow the pattern of the Examples, including indentation, a name for your solution algorithm, and an "END". Don't just use BEGIN or START. Give it an actual name, starting with a capital letter, using camelCase naming conventions like in the demos.
SOLUTION ALGORITHM:
BEGIN

calculateGreeting
  prompt for firstName
  prompt for lastName
  greeting is equal to "Hi, plus space plus firstName plus space plus lastName plus "
  display greeting
  
END

4. Checking the Solution Algorithm

  • Create a test plan for the algorithm.
    • Don't forget to fill in the VARIABLES section! That's easy to miss!
  • Have at least three test cases that test different inputs and outputs.
    • Have some interesting test cases where the first and last names are not just one simple word.
VARIABLES: firstName, lastName, output

TEST CASE 1:

  INPUT VALUES: Gunnar, Forcier
